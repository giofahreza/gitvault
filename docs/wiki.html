First setup

Applies to
Web and Android
Time
5-10 minutes
Needed
Recovery storage and optional GitHub account

Complete these steps before saving real secrets. The most important task is saving the recovery phrase outside GitVault.

  1. 1
    Open GitVault.

    Open the web app on desktop, or install the Android APK for mobile use.

  2. 2
    Choose the create-vault flow.

    Use this when you are starting fresh and do not already have a GitVault vault.

  3. 3
    Save the recovery phrase.

    Write it down or store it in a secure offline place. Do not save it inside GitVault.

  4. 4
    Create your PIN.

    Pick the PIN length you want to use. On web, you can type the PIN using the keyboard.

  5. 5
    Open the empty vault.

    Confirm that you can unlock the vault before adding passwords, notes, or 2FA codes.

  6. 6
    Set up sync if you use more than one device.

    Connect a private GitHub repository from settings, then run the first sync.

Do not skip recovery.

If you lose the recovery phrase and cannot unlock the vault, GitVault cannot read the encrypted vault data for you.

Unlock and lock

Applies to
Web and Android
Use when
Opening or leaving the vault

Unlock with PIN

  1. 1
    Open the app.

    The unlock screen appears when the vault is locked.

  2. 2
    Enter the full PIN.

    Use your keyboard on web or the on-screen keypad on mobile.

  3. 3
    Wait for verification.

    If the PIN is correct, GitVault opens the vault. If it is wrong repeatedly, wait for throttling to finish.

Enable biometric unlock

  1. 1
    Unlock with PIN first.

    Biometric unlock must be enabled from an already-unlocked vault.

  2. 2
    Open settings.

    Find the biometric unlock setting on web or Android.

  3. 3
    Turn biometric unlock on.

    Approve the browser or device prompt. Use this only on trusted devices.

Lock manually

  1. 1
    Use the lock button.

    On web, use the visible lock action before stepping away.

  2. 2
    Confirm the lock screen appears.

    The vault is protected again when the unlock screen is shown.

Passwords

Stores
Login credentials
Optional
URL, notes, group, TOTP secret

Add a password

  1. 1
    Open the Passwords feature.

    Use the password tab or menu item in the app.

  2. 2
    Select add.

    Open the add-password form.

  3. 3
    Fill the required fields.

    Add a title, username, and password. Add URL and notes when helpful.

  4. 4
    Choose or create a group.

    Pick an existing group from the dropdown or create a new group in the same control.

  5. 5
    Add TOTP if the login has 2FA.

    Paste the TOTP secret only if you want the code inside the same password record.

  6. 6
    Save and sync.

    After saving, wait for sync if you use multiple devices.

Use a saved password

  1. 1
    Search or open the group.

    Find the entry by title, username, URL, or group.

  2. 2
    Copy the needed field.

    Use copy buttons for username, password, or TOTP code instead of selecting text manually.

  3. 3
    Let clipboard auto-clear when configured.

    If clipboard auto-clear is enabled, GitVault clears copied secrets after the configured delay.

Edit or delete a password

  1. 1
    Open the password detail.

    Review the current data before changing it.

  2. 2
    Choose edit or delete.

    Use edit for normal updates. Delete only when you no longer need the entry.

  3. 3
    Save and confirm sync.

    Check another device after sync if the entry is important.

Notes

Stores

Tools
Pin, color, tags, archive, search

Create a note

  1. 1
    Open Notes.

    Use the notes tab or menu item.

  2. 2
    Select add note.

    Open the note editor.

  3. 3
    Add a title and content.

    Use notes for recovery codes, secure checklists, or private reference text.

  4. 4
    Set color, tags, or pin.

    Pin important notes and use colors or tags to make groups easier to scan.

  5. 5
    Save the note.

    Wait for sync if you need the note on another device.

Find and organize notes

  1. 1
    Use search.

    Search by title, content, tag, or other visible note details.

  2. 2
    Switch list or grid when available.

    Use the view that is easiest for the current screen size.

  3. 3
    Archive old notes.

    Archive notes you do not need daily but do not want to delete.

2FA codes

Stores
TOTP authenticator secrets
Output
Time-based 6 digit codes

Add a 2FA code manually

  1. 1
    Open 2FA Codes.

    Use the 2FA or authenticator feature in the app.

  2. 2
    Select add.

    Open the form for a new authenticator entry.

  3. 3
    Enter account details.

    Add issuer, account name, and the TOTP secret from the service you are protecting.

  4. 4
    Choose or create a group.

    Use the group dropdown to keep 2FA entries organized.

  5. 5
    Save and verify.

    Confirm that GitVault shows a live code and countdown.

Import from Google Authenticator

  1. 1
    Export from Google Authenticator.

    Use the export flow in Google Authenticator.

  2. 2
    Use the correct GitVault import path.

    On web, paste the export link. On mobile, scan the export QR code when scanner support is available.

  3. 3
    Review entries before saving.

    Make sure imported names and issuers are understandable.

Use a 2FA code

  1. 1
    Open the 2FA entry.

    Wait for the countdown if the current code is nearly expired.

  2. 2
    Copy the code.

    Paste it into the service login form before the timer resets.

SSH entries

Stores
SSH host credentials
Best on
Android for terminal workflows

Add an SSH entry

  1. 1
    Open SSH.

    Use the SSH feature in the app.

  2. 2
    Select add.

    Open the SSH entry form.

  3. 3
    Enter connection details.

    Add host, username, and port. Use a clear title if the host name is not obvious.

  4. 4
    Add credential data.

    Save the password or key information required for your workflow.

  5. 5
    Save and sync.

    Run sync before expecting the SSH entry on another device.

Use an SSH entry

  1. 1
    Open the saved entry.

    Review host, username, and port before connecting.

  2. 2
    Copy fields or launch terminal when available.

    On Android, use the terminal experience when supported by the app build and device.

  3. 3
    Lock when finished.

    Lock GitVault after working with server credentials.

Sync and devices

Storage
Private GitHub repository
Data sent
Encrypted vault files only

Set up GitHub sync

  1. 1
    Create an empty private repository.

    Use your normal GitHub account. Keep the repository private.

  2. 2
    Create a fine-grained token.

    Grant only the repository access GitVault needs, such as metadata read and contents read/write.

  3. 3
    Open GitVault settings.

    Go to sync settings and paste the repository and token details.

  4. 4
    Run the first sync.

    Wait for a successful sync state before closing the app.

Add another device

  1. 1
    Install or open GitVault on the new device.

    Use web for browser access or Android APK for mobile access.

  2. 2
    Use restore or link flow.

    Follow the app flow for connecting to an existing vault.

  3. 3
    Connect the same GitHub repository.

    Use the same sync storage so devices share encrypted vault files.

  4. 4
    Open connected devices.

    After sync, check settings to confirm the device appears.

Sync daily changes

  1. 1
    Save the change.

    Edit or add the entry on the current device.

  2. 2
    Wait for sync.

    Do not immediately switch devices if sync is still running.

  3. 3
    Open the other device and sync.

    If changes do not appear, manually trigger sync or reopen the app.

Recovery

Priority
High
Needed
Recovery phrase and sync access

Restore on a new device

  1. 1
    Open GitVault on the new device.

    Choose the restore or existing-vault flow.

  2. 2
    Enter the recovery information.

    Use the recovery phrase you saved during first setup.

  3. 3
    Connect sync storage.

    Use the same private GitHub repository and valid token.

  4. 4
    Verify entries.

    Check passwords, notes, 2FA codes, and settings before relying on the device.

If a device is lost

  1. 1
    Assume copied secrets may be exposed.

    Rotate important passwords, especially accounts used on that device.

  2. 2
    Rotate the GitHub token.

    Create a new token and stop using the old one.

  3. 3
    Review connected devices.

    Open GitVault on a trusted device and check device status after sync.
